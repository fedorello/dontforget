# DontForget — Concept Document

## Идея в одной фразе

Приложение, которое помнит за тебя всё важное — лекарства, дела, покупки — и говорит об этом просто, вовремя и с заботой.

---

## Проблема

Человек держит в голове слишком много: когда пить таблетки, что купить, что сделать. Особенно сложно с лекарствами и БАДами — непонятные названия, режимы приёма, взаимодействия. Ещё сложнее когда ты следишь не только за собой, но и за близкими.

---

## Решение

Минималистичное мобильное приложение с тремя модулями:
1. **Лекарства и БАДы** — умные напоминания с AI-анализом
2. **Покупки** — множество списков
3. **TODO** — простые задачи

Главный принцип: **минимум усилий от пользователя, максимум заботы от приложения**.

---

## Платформы и стек

- **Frontend:** Flutter (Android первично, iOS — следующий этап)
- **Backend:** Python / FastAPI, задеплоен на **Railway.app**
- **AI:** OpenRouter, модель `x-ai/grok-4.1-fast`
- **Конфигурация:** `.env` / `.env.example`
- **База данных:** PostgreSQL (Railway managed)
- **Хранилище фото:** S3-совместимое (Cloudflare R2 или Railway volume)
- **Push-уведомления:** Firebase Cloud Messaging (FCM)

## Модуль 1: Лекарства и БАДы

### 1.1 Добавление препарата

Пользователь открывает раздел и нажимает "+". Приложение предлагает:
- **Сфотографировать** коробку, инструкцию, блистер (можно несколько фото)
- Или ввести название вручную (поиск по базе)

AI (через OpenRouter / grok-4.1-fast) анализирует фото и извлекает:
- Название препарата (торговое и МНН)
- Действующее вещество(а)
- Дозировка в упаковке
- Показания (упрощённо)
- Противопоказания
- Побочные эффекты (топ-5, простым языком)
- Рекомендуемый режим приёма

Вся информация сохраняется в **Базе Знаний** приложения. Если препарат уже был добавлен другим пользователем — используется кэшированная запись (с возможностью уточнения).

### 1.2 Профили пользователей

В приложении можно создать несколько профилей:
- Я
- Жена / Муж
- Ребёнок
- Мама / Папа
- Любой другой человек

Каждый профиль содержит:
- Имя и фото (опционально)
- Дата рождения (возраст вычисляется автоматически)
- Пол
- Вес (опционально, для расчёта доз)
- Хронические заболевания / особенности здоровья (в свободной форме или из списка)
- Аллергии
- Текущий список активных препаратов (формируется автоматически)

### 1.3 Персональные рекомендации

После распознавания препарата приложение выбирает профиль (или спрашивает — для кого) и показывает **карточку рекомендаций**:

```
Омега-3 / Рыбий жир

Что делает:
Поддерживает работу сердца и мозга, снижает воспаление.

Рекомендации для Тео (35 лет, муж.):
- Доза: 1 капсула (1000 мг)
- Когда: утром, во время еды
- Курс: 3 месяца, затем перерыв 1 месяц

Совместимость:
✓ Хорошо сочетается с Магнием и Витамином D
⚠ С разжижителями крови — проконсультируйся с врачом

Побочные эффекты (редко):
рыбный привкус, лёгкое расстройство желудка
```

Пользователь может принять рекомендации или скорректировать:
- Изменить дозу
- Изменить время / кратность
- Изменить продолжительность курса
- Добавить заметку

### 1.4 Анализ совместимости

При добавлении нового препарата система автоматически проверяет его против **всех активных препаратов** выбранного профиля:

- **Зелёный:** совместим, возможен синергетический эффект
- **Жёлтый:** принимать в разное время / с осторожностью
- **Красный:** несовместимость, рекомендуется не совмещать

Предупреждение показывается до запуска напоминаний. Пользователь может принять риск осознанно (с подтверждением).

### 1.5 Напоминания

После настройки запускаются push-уведомления:

**Формат уведомления:**
```
Время принять: Омега-3
1 капсула · во время еды
→ Для сердца и мозга
```

Звук: мягкий, ненавязчивый (не будильник).

Внутри приложения — экран "Сейчас нужно принять" с кнопками:
- **Принял** (зелёная, большая)
- **Отложить на 15 мин**
- **Пропустить сегодня**

История приёмов сохраняется. Можно посмотреть статистику соблюдения курса.

### 1.6 Завершение курса

За 3 дня до конца курса — напоминание: "Курс Омега-3 заканчивается. Продолжить?" Пользователь может продлить или завершить. После завершения препарат переходит в архив (не активный).

### 1.7 Уведомление об окончании упаковки

Пользователь при создании указывает количество таблеток/капсул в упаковке. Приложение считает остаток и предупреждает за 5-7 дней: "Осталось ~10 таблеток Омега-3. Стоит купить новую упаковку."

## Модуль 2: Списки покупок

Простой и быстрый модуль. Главное — скорость добавления.

- Несколько именованных списков (Продукты, Аптека, IKEA, Стройматериалы и т.д.)
- Быстрое добавление: поле в одно касание, Enter — следующий пункт
- Голосовой ввод (через системный микрофон)
- Группировка по категориям (авто или вручную)
- Совместный список: поделиться ссылкой → второй человек видит и отмечает в реальном времени
- Когда пункт куплен — зачёркивается (не удаляется сразу)
- Кнопка "Очистить купленное"
- Повторяющиеся списки: "Стандартная закупка" → одним нажатием создать новый список из шаблона

**Интеграция с лекарствами:** когда препарат заканчивается, приложение предлагает добавить его в список покупок "Аптека" автоматически.

## Модуль 3: TODO

Простой список задач без лишней сложности.

- Задача = текст + опциональная дата/время напоминания
- Подзадачи (один уровень вложенности)
- Теги / цвета для категоризации
- Фильтр: Сегодня / На этой неделе / Все / Выполненные
- Повторяющиеся задачи (ежедневно, по дням недели, ежемесячно)
- Приоритет: обычный / важный

Намеренно **без:** проектов, досок Kanban, временных шкал — это не Notion и не Jira.

## UX / Дизайн

### Принципы
- **Тихий помощник:** приложение не кричит, не давит, не засыпает уведомлениями
- **Один экран — одно действие:** каждый шаг понятен без инструкций
- **Простой язык:** никаких медицинских терминов без объяснений
- **Тепло и забота:** иллюстрации, мягкие цвета, округлые формы

### Цветовая палитра
- Фон: тёплый белый / очень светло-бежевый
- Акцент: мягкий зелёный (здоровье), тёплый голубой (спокойствие)
- Предупреждения: тёплый оранжевый (не красный — красный пугает)
- Критические: мягкий красный только для несовместимости

### Навигация
Нижнее меню: 4 иконки
- Сегодня (главный экран)
- Лекарства
- Покупки
- Задачи

### Онбординг
5 простых экранов при первом запуске:
1. "Привет! Я помогу не забывать важное"
2. "Создай свой профиль" (имя, возраст, пол)
3. "Добавь первое лекарство или БАД"
4. "Настрой уведомления"
5. "Всё готово!"

## Backend API (FastAPI / Railway)

### Ключевые эндпоинты

```
POST   /api/analyze/photo          — анализ фото препарата через AI
GET    /api/medicines/search        — поиск в базе знаний
POST   /api/medicines/compatibility — проверка совместимости
POST   /api/profiles               — создание профиля
GET    /api/profiles/{id}/schedule  — расписание приёмов
POST   /api/reminders              — создание напоминания
PATCH  /api/reminders/{id}/taken   — отметить как принятое
GET    /api/reminders/today         — список на сегодня
POST   /api/lists                  — создание списка покупок
POST   /api/todos                  — создание задачи
POST   /api/push/register          — регистрация FCM токена
```

### AI интеграция (OpenRouter)

```python
# .env
OPENROUTER_API_KEY=...
OPENROUTER_MODEL=x-ai/grok-4.1-fast
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1

DATABASE_URL=postgresql://...
FCM_SERVER_KEY=...
S3_BUCKET=...
S3_ENDPOINT=...
S3_ACCESS_KEY=...
S3_SECRET_KEY=...
```

Запросы к AI:
1. **Распознавание фото** — vision-запрос с изображением(ями) упаковки
2. **Генерация рекомендаций** — с учётом профиля пользователя
3. **Анализ совместимости** — список активных препаратов + новый

### Кэширование
Результаты анализа препаратов кэшируются в БД по хэшу названия/штрихкода. Повторный запрос не идёт к AI — используется кэш. Это экономит токены и ускоряет работу.

## Уведомления — детали

- Уведомления приходят за 5 минут до назначенного времени
- Если не отмечено "принял" в течение 30 минут — мягкое повторное напоминание (одно)
- Режим "Тихий час" — уведомления не приходят в заданный интервал (например, 23:00–7:00), переносятся на начало следующего окна
- Краткое содержание уведомления: имя препарата, доза, зачем (одна фраза)
